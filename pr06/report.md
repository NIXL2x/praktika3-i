# ПР №6. Анализ сетевого трафика: tcpdump и Wireshark

## 1. Разбор строки tcpdump

Строка: `15:40:07.999978 IP6 ::1 > ::1: ICMP6, echo reply, id 2372, seq 1, length 64`

| Поле | Значение | Что означает |
|------|---------|-------------|
| **15:40:07.999978** | Время | Точное время перехвата пакета. |
| **IP6** | Протокол | Сетевой протокол IPv6. |
| **::1** (первый) | Источник | IPv6-адрес отправителя. |
| **>** | Направление | Указывает направление движения трафика. |
| **::1** (второй) | Назначение | IPv6-адрес получателя. |
| **ICMP6, echo reply** | Тип данных | Протокол управляющих сообщений, тип: эхо-ответ. |
| **id 2372** | ID | Идентификатор процесса. |
| **seq 1** | Номер | Порядковый номер пакета. |
| **length 64** | Длина | Размер полезной нагрузки в байтах. |

## 2. Перехваченные данные HTTP

**Что видит злоумышленник:**
В теле POST-запроса открытым текстом видна строка параметров авторизации: `username=admin&password=supersecret123`.

**Кто в реальной сети может так же прочитать этот трафик:**
1. Пользователи в той же локальной сети (LAN) при проведении атак MITM.
2. Администраторы или атакующие в публичных Wi-Fi сетях.
3. Интернет-провайдеры (.
4. Владельцы скомпрометированных маршрутизаторов на пути трафика.

## 3. Follow TCP Stream

**Что видит злоумышленник, перехвативший этот трафик:**
Полный текстовый диалог между клиентом и сервером в хронологическом порядке: метод и URL запроса (`POST /login HTTP/1.1`), заголовки (User-Agent, Host), переданные учетные данные (`user=bob&pass=qwerty`) и ответ сервера.

## 4. HTTP vs HTTPS — сравнение

| Что наблюдаем | HTTP | HTTPS |
|---|---|---|
| **IP-адреса** | Видны в открытом виде | Видны в открытом виде |
| **Заголовки** | Видны полностью | Зашифрованы внутри TLS-сессии |
| **Тело запроса** | Видно открытым текстом | Зашифровано алгоритмами AES/ChaCha20 |
| **Логин/пароль** | Видны в исходном текстовом виде | Зашифрованы внутри TLS-сессии |
| **Имя домена** | Видно в заголовке Host | Видно на этапе SNI, далее скрыто |

## 5. DNS-утечка при HTTPS

![Перехват DNS-запросов](screens/dns-leak.png)

**Какую информацию получает злоумышленник из DNS-трафика:**
Протокол DNS (порт 53) работает без шифрования. Из перехваченных строк вида `A? google.com` злоумышленник видит точные доменные имена сайтов, которые запрашивает пользователь, что позволяет отслеживать его активность.

## 6. Base64 — это не шифрование

Результат выполнения команды: `vasya:MyPassword2024`

**Почему кодирование Base64 не является средством защиты информации:**
Base64 — это алгоритм обратимого преобразования данных в ASCII-строку для передачи в текстовых протоколах. Он не использует секретный ключ. Любой наблюдатель может декодировать строку Base64 обратно в исходный текст стандартными средствами.

## 7. tshark vs tcpdump

| Инструмент | Когда использовать |
|---|---|
| **tcpdump** | Быстрый перехват сырых пакетов в консоли (CLI) на серверах без GUI при дефиците ресурсов. |
| **tshark** | Анализ и парсинг pcap-файлов в консоли с использованием дисплейных фильтров, автоматизация. |
| **Wireshark** | Детальный ручной разбор инцидентов в графическом интерфейсе, чтение сессий. |

## 8. Каналы утечки через сеть

**Какие меры защиты закрывают сетевой канал утечки:**
1. Использование защищенных протоколов с шифрованием TLS/SSL (HTTPS, SSH, SFTP).
2. Шифрование DNS-запросов с помощью технологий DoH (DNS over HTTPS) или DoT (DNS over TLS).
3. Применение VPN для шифрования всего транзитного трафика от узла до шлюза.
4. Сегментация сети (VLAN) и настройка правил межсетевого экрана.

## Контрольные вопросы

**78. Что такое promiscuous mode и зачем он нужен снифферу?**
Promiscuous mode — это режим работы сетевой карты, при котором она отключает фильтрацию по MAC-адресу назначения и передает операционной системе абсолютно все пакеты, поступающие на физический порт, а не только те, что адресованы конкретно этому узлу, бродкастовые или мультикастовые. Снифферу этот режим необходим для перехвата чужого транзитного трафика в пределах одного сетевого сегмента.

**79. Чем отличается capture filter от display filter в Wireshark?**
* **Capture filter (Фильтр захвата):** Задается до начала перехвата. Пакеты, не подошедшие под критерий, отбрасываются ядром ОС на лету и не записываются в память или pcap-файл. Изменить его после старта нельзя.
* **Display filter (Фильтр отображения):** Применяется к уже захваченному и сохраненному массиву данных. Он скрывает или отображает пакеты на экране для удобства анализа, не удаляя их из исходного pcap-файла. Его можно менять в любой момент без перезапуска захвата.

**80. Что такое Follow TCP Stream и для чего используется?**
Функция «Follow TCP Stream» в Wireshark автоматически собирает разрозненные сетевые пакеты конкретного TCP-соединения, очищает их от технических заголовков (IP, TCP) и объединяет ASCII-данные в единый текстовый диалог в хронологическом порядке. Используется для анализа прикладного уровня: чтения переписки, просмотра структуры HTTP-запросов и ответов, а также извлечения передаваемых файлов.

**81. Почему Base64 в HTTP Basic Auth не является защитой? Что нужно использовать вместо?**
Base64 является симметричным и полностью обратимым алгоритмом кодирования, а не шифрования. В нем отсутствует секретный ключ, поэтому любая третья сторона, перехватившая заголовок `Authorization: Basic...`, может мгновенно декодировать его в исходный вид «логин:пароль» стандартной системной утилитой. Вместо этого необходимо использовать криптографическую защиту транспортного уровня — протокол TLS/SSL (HTTPS), который шифрует всю сессию, включая заголовки.

**82. Какие данные утекают даже при использовании HTTPS? Как это исправить?**
При использовании HTTPS сторонний наблюдатель видит:
1. IP-адреса источника и назначения.
2. Объем передаваемого трафика и время активности.
3. Запрашиваемое доменное имя

**83. Сисадмин говорит: «у нас свитч, а не хаб — значит сниффинг невозможен». Он прав?**
Сисадмин не прав. Коммутатор действительно направляет трафик только на тот порт, где находится адресат (на основе таблицы MAC-адресов), что защищает от пассивного сниффинга. Однако злоумышленник может применить методы активного перехвата данных: ARP-spoofing, MAC-flooding, либо использовать легитимную функцию зеркалирования портов, если коммутатор управляемый и скомпрометирован.

**84. Напишите фильтр tcpdump который перехватывает только DNS-запросы (UDP порт 53) от конкретного IP 192.168.1.50.**
udp dst port 53 and src host 192.168.1.50
